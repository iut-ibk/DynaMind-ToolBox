Working with Modules¶

Set Successor Mode¶

The successor mode forces a split of the data stream. This means that a new data base is created for the down stream modules. To switch set the successor model right click on the module to open the context menu and select force successor mode

Tip

Successor modes are useful to debug a simulation since it allows to export the database at this point in the simulation (see Export Simulation Database )

Export Simulation Database¶

To export the simulation database right click on the module to open the context menu and select export data at out port. The simulate database is exported as SpatiaLite database and can open with standard GIS software e.g. QGIS.

Note

This option is only available for modules that use the GDAL database backend.
